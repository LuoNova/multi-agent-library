# 图书馆管理系统 · 前端（Vue 3）

与仓库根目录 **Spring Boot 后端**（默认 `http://localhost:8080`）通过 **开发代理** 联调。

## 环境要求

- **Node.js**：建议 **20.16+**（当前 `package.json` 已固定 **Vite 5**，避免 Vite 8 对 Node 20.19+ 的硬性要求）。
- 后端已启动且端口为 **8080**。

## 安装依赖

```bash
cd frontend
npm install
```

## 本地开发

```bash
npm run dev
```

浏览器访问终端里提示的地址（一般为 **`http://localhost:5173`**）。

- 开发服务器已将 **`/api`**、**`/test`** 代理到 `http://localhost:8080`，无需在前端配置 CORS。
- **假登录**：先打开 **`/`** 选择读者端或管理端，再在 **`/login/user`** 或 **`/login/admin`** 登录；任意用户名/密码即可。用户名为**纯数字**时，会作为各页面默认的「用户 ID」写入本地存储（`lib_user_id`）。登录时会写入 **`lib_portal_mode`**（`user` / `admin`），与 **`lib_fake_token`** 绑定：读者端会话不能访问 `/a/*`，管理端不能访问 `/u/*`；**`/login`** 会重定向到 **`/`**。退出后回到 **`/`** 重新选端。

## 生产构建

```bash
npm run build
```

产物在 **`frontend/dist/`**，可将该目录部署到任意静态站点（Nginx、Spring 静态资源等）。部署到与后端不同源时，需配置反向代理或后端 **CORS**，使浏览器能访问 `/api`。

## 功能范围（第一版）

前端按 **读者服务**（`/u/*`）与 **馆员/运维管理**（`/a/*`）拆分；**必须先选端再登录**，两端路由互不贯通。旧路径（如 `/borrow`）会 **302 重定向** 到新地址。

| 路由 | 说明 |
|------|------|
| `/` | 公开：选择读者端 / 管理端，进入对应登录页 |
| `/login/user` | 读者端登录（成功后仅可访问 `/u/*`） |
| `/login/admin` | 管理端登录（成功后仅可访问 `/a/*`） |
| `/login` | 重定向到 `/` |
| `/u/...` | 读者端业务（需读者端登录） |
| `/a/...` | 管理端业务（需管理端登录） |
| `/borrow` 等 | 兼容旧书签，重定向到对应 `/u/*` 或 `/a/*` |

更完整的接口说明见仓库根目录 **`API接口文档-前后端对接.md`**。  
**中文标签、枚举释义、结果解析约定**见 **`docs/前端字段与接口约定.md`**（与 `src/utils/labels.ts`、`DataResultPanel` 组件保持一致）。
