## 功能1：**智能借书与资源调度（Smart Borrowing）**

### 功能描述

用户发起借书请求后，系统通过多智能体协作自动选择最优策略：

- **本地借阅（LOCAL_LOAN）**：目标馆有库存时立即借出
- **跨馆调拨（TRANSFER_PROVIDE）**：目标馆无库存但其他馆有，自动协商调拨并预留24小时
- **预约排队（RESERVATION）**：全馆无库存时创建预约，书归还后自动优先分配

系统内置权限检查（最大借阅数、重复借书限制），确保业务合规。

### 参与智能体

| 智能体              | 角色                 | 职责                                             |
| :------------------ | :------------------- | :----------------------------------------------- |
| UserDemandAgent     | Manager（协调者）    | 接收请求、权限检查、策略选择、调拨执行、预约管理 |
| InventoryAgent-LibN | Contractor（参与者） | 评估本地库存、计算投标评分（距离+压力）、响应CFP |

### 业务流程（时序）

1. **权限检查**：UserDemandAgent检查用户状态、最大借阅数、是否已借该书
2. **策略选择**：
   - **本地优先**：检查`preferredLibraryId`是否有`AVAILABLE`副本 → 直接借书（策略A）
   - **跨馆协商**：本地无库存 → 广播CFP → 收集投标 → 选择最优馆调拨（策略B）
   - **预约兜底**：全馆无库存 → 创建`tb_book_reservation`记录（策略C）
3. **执行与预留**：
   - 策略A：创建借阅记录（`BORROWING`），副本状态改为`BORROWED`
   - 策略B：执行调拨（`tb_book_transfer`），创建预留记录（`RESERVED`，24小时有效）
   - 策略C：创建预约记录（`PENDING`，3天有效）
4. **结果返回**：返回对应策略的执行结果

### 核心算法

**投标评分公式**（仅策略B使用）：

```plain
score = 80 - (distance(km) × 10) - (pending_reservation_count × 5)
```

### 输入参数

| 字段               | 类型 | 必填 | 说明         | 示例        |
| :----------------- | :--- | :--- | :----------- | :---------- |
| userId             | Long | 是   | 用户ID       | 1           |
| biblioId           | Long | 是   | 书目ID       | 1           |
| preferredLibraryId | Long | 是   | 期望取书馆ID | 1（理科馆） |

**请求示例**：

JSON

```json
{
  "userId": 1,
  "biblioId": 1,
  "preferredLibraryId": 1
}
```

### 输出结果（按策略区分）

**注意**：`strategy`字段标识实际执行的策略，不同策略返回的字段集合不同。

| 字段            | 类型    | 说明                                        | 适用策略                       |
| :-------------- | :------ | :------------------------------------------ | :----------------------------- |
| taskId          | String  | 任务唯一标识                                | 所有                           |
| status          | String  | SUCCESS / FAILED                            | 所有                           |
| strategy        | String  | LOCAL_LOAN / TRANSFER_PROVIDE / RESERVATION | 所有                           |
| message         | String  | 详细提示信息                                | 所有                           |
| copyId          | Long    | 分配的副本ID                                | LOCAL_LOAN / TRANSFER_PROVIDE  |
| libraryId       | Long    | 取书馆ID（本地借阅时）                      | LOCAL_LOAN                     |
| sourceLibraryId | Long    | 来源馆ID（调拨时）                          | TRANSFER_PROVIDE               |
| targetLibraryId | Long    | 目标馆ID（调拨时）                          | TRANSFER_PROVIDE               |
| reserveHours    | Integer | 预留时长（小时）                            | TRANSFER_PROVIDE / RESERVATION |
| reservationId   | Long    | 预约记录ID                                  | RESERVATION                    |
| pickupLibraryId | Long    | 预约取书馆ID                                | RESERVATION                    |

### 不同场景的响应示例

#### 场景A：本地直接借阅（目标馆有库存）

**条件示例**：`preferredLibraryId=1`（理科馆），且理科馆有`AVAILABLE`副本

**响应示例**：

```json
{
  "taskId": "TASK-1772004850949",
  "status": "SUCCESS",
  "strategy": "LOCAL_LOAN",
  "copyId": 1002,
  "libraryId": 1,
  "message": "借书成功，请在30天内归还"
}
```

#### 场景B：跨馆调拨+预留（目标馆无，其他馆有）

**条件示例**：理科馆0本，文科馆2本

**响应示例**：

```json
{
  "taskId": "TASK-1772004850950",
  "status": "SUCCESS",
  "strategy": "TRANSFER_PROVIDE",
  "sourceLibraryId": 2,
  "targetLibraryId": 1,
  "copyId": 1001,
  "reserveHours": 24,
  "message": "调拨已完成，已为您预留24小时，请尽快取书"
}
```

#### 场景C：预约排队（全馆无库存）

**条件示例**：理科馆0本，文科馆0本（全部被借出）

**响应示例**：

```json
{
  "taskId": "TASK-1772004850951",
  "status": "SUCCESS",
  "strategy": "RESERVATION",
  "reservationId": 5,
  "pickupLibraryId": 1,
  "reserveHours": 72,
  "message": "全馆暂无库存，已为您预约（3天内有效），书到货后将优先通知您取书"
}
```

#### 失败场景：权限不足

**条件示例**：用户已达最大借阅数，或已借阅该书

**响应示例**：

```json
{
  "taskId": "TASK-1772004850952",
  "status": "FAILED",
  "message": "权限检查失败: 已达最大借阅数限制（当前10/10）"
}
```

### 数据库变更

**策略A（本地借阅）**：

- **写入**：`tb_book_borrow`（状态`BORROWING`），`tb_book_copy`（状态改为`BORROWED`，`local_borrow_count`+1）
- **更新**：`tb_user`（`current_borrow_count`+1）

**策略B（跨馆调拨）**：

- **写入**：`tb_book_transfer`（调拨记录），`tb_book_borrow`（状态`RESERVED`，24小时过期时间）
- **更新**：`tb_book_copy`（`library_id`改为目标馆，状态`RESERVED`），`tb_library_biblio_stats`（两馆库存同步增减），`tb_user`（`current_borrow_count`+1）

**策略C（预约）**：

- **写入**：`tb_book_reservation`（状态`PENDING`，3天过期时间）
