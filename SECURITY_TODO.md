# 安全相关待办（暂不实现）

以下项在业务演示阶段可暂缓，上线前应逐项处理。

1. **认证与授权**：`POST /api/fault/report` 的 `reportUserId` 应从登录态注入；工单列表与 `PATCH /api/fault/{id}/status` 应限制为管理员或运维角色。
2. **设备状态接口**：`PATCH /api/equipment/{id}/status` 当前无权限控制，存在越权改状态风险。
3. **JADE / ACL**：智能体消息无签名与加密，仅适用于内网演示。
4. **输入与审计**：对工单标题、描述等字段做长度与内容策略；重要状态变更写审计日志。

详见《后续开发计划与待确认事项》阶段 D。
